学习目标

本章把原理图视为一套有语法的工程语言。学完后，你应能通过位号、引脚号和网络名追踪连接，识别信号方向与默认状态，并能先按功能块阅读，再深入到单个元件。

  • 解释常见位号前缀以及它们不能告诉你的信息；
  • 区分导线交点、网络标签、总线和电源符号；
  • 识别高有效、低有效、输入、输出、双向和高阻态；
  • 说明 GND 是参考与返回网络，而不是电流消失处；
  • 正确区分 R1 的四条主要电源轨；
  • 沿 U1、R14 与 J2 追踪一条真实信号路径。

2.1 位号：R、C、L、D、U、J、Y 与 TP

位号（Reference Designator）是原理图和 PCB 中某个元件实例的唯一名称。前缀通常表示元件类别，数字区分同类实例。位号能回答“是哪一个”，但不能单独回答“阻值是多少”“型号是什么”或“放在哪里”。

R电阻R1/R2 为 USB-C 配置通道（Configuration Channel，CC）下拉；R14–R23 为外部信号串联电阻
C电容C6–C8 为 U1 高频去耦；C10 与 R12 构成复位 RC

同一个元件在原理图、PCB、BOM、CPL 和装配图中必须保持相同位号。R1 的发布审核确认 48/48 个原理图元件映射到 PCB；J2 和 J3 虽然在 PCB 与 CPL 中存在，但标记为 DNP，不进入拟采购装配的 BOM。

2.2 导线、连接点、标签、总线与电源符号

导线（Wire）
原理图中的连接线，表示其端点在电气上应属于同一网络；线条长度通常不代表 PCB 走线长度。
连接点（Junction）
明确表示多条导线在交点电气相连。仅仅视觉交叉不一定连接，必须遵守所用 EDA 的连接点规则。
网络标签（Net Label）
给网络命名，使相距较远的同名节点相连并提高可读性。例如所有 3V3_SYS 标签属于同一网络。
总线（Bus）
用于整理一组相关信号的图形和命名方式。总线本身不自动创造正确连接，仍要核对每根成员信号。
电源符号
常用于全局或局部电源网络。符号朝上、朝下只是绘图习惯，不代表电流只能按纸面方向流动。

常见错误：看到两根线交叉就认为它们相连，或者认为两个文字相似的标签必然相同。3V3_SYS、3V3_SW 与 GPIO3_MCU 是完全不同的网络；一个字符的差异就可能改变电气结果。

阅读时不要只沿线移动眼睛。每到一个元件引脚，记录“位号—引脚号—网络名”，这三项共同建立可复核的连接证据。

2.3 高有效、低有效、输入、输出与双向信号

高有效（Active-High）表示逻辑高电平触发功能；低有效（Active-Low）表示逻辑低电平触发功能。低有效常用名称上的横线、前缀 n、后缀 _N 或约定名称表示，但不能只凭字体猜测，必须查数据手册。

R1 的 U4、U5 为高有效负载开关：EN_5V 或 EN_3V3 拉高时，设计意图是开启相应电源。R9、R10 各用 100 kΩ 下拉，在 U1 复位或引脚高阻时把使能保持为低。NRST 则是 MCU 的低有效复位输入，拉低会使 U1 保持复位。

信号方向描述
类型含义R1 示例
输入接收外部状态，不应主动驱动该网络U1 的 UART_RX_MCU
输出主动产生逻辑电平U1 的 EN_5V、STATUS_LED
双向可在不同时间输入或输出，必须避免双方同时反向驱动8 路 GPIO
高阻态（Hi-Z）引脚暂不主动拉高或拉低R1 的安全启动目标要求外部 GPIO 和 UART TX 在配置前为高阻

“高阻”不等于“电压必为 0”。高阻节点可能被外部上拉、下拉、漏电或电容保持在某个电压，所以安全设计常增加明确的偏置电阻。R1 的上电安全状态是设计要求与预期行为，尚需固件和实物实验确认。

2.4 地是参考与返回网络，不是魔法汇点

原理图上重复出现 GND 符号，是为了减少长线并说明这些节点属于共同的地网络。GND 定义了本板的 0 V 参考，并为电源和信号电流提供返回路径；它不会使电荷或能量“消失”。

例如 U1 输出高电平点亮 D1 时，传统电流路径为 3V3_SYS → U1 输出级 → R13 → D1 → GND → U1 电源返回。原理图可能把多个 GND 符号画在不同位置，但 PCB 必须用铜把它们连接，并控制返回路径的阻抗与回路面积。

工程后果：R1 与 USB 主机、目标设备共地且不隔离。若目标系统由另一个电源供电，连接 J2 前应先确认地电位兼容；否则信号线或 USB 屏蔽/地可能承担意外均衡电流。

R1 最终制造数据包含 173 条显式 GND 走线、32 个地缝合/树状过孔，以及 17 个与 GND 相连的静态铜填充区域。这说明 GND 在 PCB 上是具体铜网络，而不是一个抽象的“删除电流”符号。

2.5 电源轨以及 VBUS_IN、3V3_SYS、3V3_SW 与 5V_SW 的区别

电源轨（Power Rail）是向一组负载供电的命名网络。即使两个电源轨的标称电压相同，也可能因来源、开关状态、限流和用途不同而必须分开。

R1 的主要电源网络
网络来源用途状态说明
VBUS_INJ1 的 USB VBUSU3 输入、U4 输入及 USB 前端参考设计标称约 5 V；真实主机、线缆压降待测
3V3_SYSU3 SY8089 降压型开关变换器（Buck）U1、逻辑电路及 U5 输入，始终开启的系统电源反馈设计计算为 3.318 V；实物电压待测
5V_SWU4 从 VBUS_IN 受控输出J2 的目标侧 5 V 电源设计限流约 0.402 A，默认关闭；阈值与压降待测
3V3_SWU5 从 3V3_SYS 受控输出J2 的目标侧 3.3 V 电源设计限流约 0.252 A，默认关闭；阈值与压降待测

3V3_SYS 与 3V3_SW 标称都是 3.3 V，但 U5 关闭时必须彼此隔离；若在原理图中误用同一网络名，负载开关会被旁路，默认关闭和限流功能都失效。同理，5V_SW 是电源输出，不是可选择的 5 V GPIO 逻辑电平。

2.6 阅读元件引脚号与封装视图

符号旁的引脚号应对应实体封装上的焊盘编号。引脚名称描述功能，编号描述物理位置；两者不能互换。U1 的 PA6 是功能名，LQFP-32 的 12 是封装脚号，最终映射为 GPIO0_MCU。

算例 2-1：追踪 GPIO0。

  1. 查 U1：封装 12 脚为 PA6，可复用为 TIM3_CH1。
  2. 该脚连接网络 GPIO0_MCU。
  3. R14 的 1 脚接 GPIO0_MCU，2 脚接 GPIO0_PWM0。
  4. GPIO0_PWM0 到 J2 的 5 脚。
  5. 因此 J2-5 可作为普通 GPIO0 或 PWM0，但不能同时由两个互相冲突的功能占用。

封装视图可能采用顶视图、底视图或器件面视图。连接器尤其容易因镜像而出错。J1 是可翻转 USB-C 插座：A6/B6 都属于 D+，A7/B7 都属于 D−，A5 与 B5 分别为 CC1、CC2。读图时要同时确认资料的观察方向、Pin 1 标记和板边朝向。

R1 发布审核确认 172/172 个原理图引脚与 PCB 焊盘按“元件身份—引脚号—网络”一致，但这仍不免除收板后的方向与焊接检查。

2.7 不连接引脚、保留引脚与有意不装配的元件

不连接（No Connect，NC）表示设计者有意让某引脚悬空，而不是忘记连线。EDA 中应使用明确的 No-Connect 标记，使审核者和 ERC 能区分“有意”与“遗漏”。R1 有 9 个经过核对的有意未连接引脚；J2 的 16 脚也保留未连接，供未来功能考虑。

保留引脚通常由数据手册规定处理方式：可能要求悬空、接地或禁止连接。不能因为名字写着 Reserved 就随意接到测试点。

DNP（Do Not Populate，不装配）与 NC 不同。DNP 元件在 PCB 上有焊盘和位置，但当前装配不放器件。R1 的 J2、J3 在 PCB 与 CPL 中存在，原理图属性为不加入 BOM，最终采购 BOM 也排除它们。

三种“没有连接/没有器件”状态
状态发生在哪里应怎样记录

2.8 按功能块阅读，而不是逐线猜测

面对完整原理图时，先找输入、输出与功能边界，再深入单线。逐线从左上角读到右下角容易失去电源来源和控制关系，也容易把相邻但不相连的电路误认为一组。

  1. 找连接器：J1 从主机带来 VBUS、GND、CC 和 USB 数据；J2 输出目标信号与电源；J3 用于调试。
  2. 找电源轨：从 VBUS_IN 追踪到 U3、U4，再从 3V3_SYS 追踪到 U1、U5。
  3. 找控制中心：U1 接收 USB，控制 GPIO/PWM/UART、状态 LED 和两路电源使能。
  4. 找保护边界：U2 位于 USB 入口；R3/R4 位于 USB 数据路径；R14–R23 位于外部信号路径。
  5. 找时钟、复位与启动：Y1/C11/C12、R12/C10、R11 分别建立时钟、复位和默认启动状态。
  6. 最后核对每个引脚：确认块间接口、方向、默认状态和网络名；串行线调试（Serial Wire Debug，SWD）连接也应单独核对。

检查点：每读完一个功能块，尝试回答四个问题：能量从哪里来？信息从哪里来？由谁控制？故障时会影响哪些相邻块？

2.9 练习：把 USB GPIO 原理图划分为功能块

目的：在不依赖布局位置的情况下，用原理图连接关系划分系统，并建立块间接口表。

资料：R1 最终原理图 PDF或 EasyEDA 工程，只读查看；彩色笔或批注工具；本章电源轨表。

步骤：

  1. 圈出 J1、J2、J3，并标出每个连接器带来的电源、地与信号。
  2. 用一种颜色追踪 VBUS_IN，另一种颜色追踪 3V3_SYS。
  3. 把元件分入以下候选块：USB-C/CC、USB 数据保护、Buck、5 V 开关、3.3 V 开关、MCU、时钟/复位/状态、外部接口、调试。
  4. 对每个块填写“输入—输出—控制—返回路径—失效影响”。
  5. 任选 GPIO0，按“U1 引脚—MCU 侧网络—串联元件—外部网络—连接器脚”写出完整路径。
  6. 与事实表核对：不要把提案中的外部 GPIO ESD 阵列或负载开关故障输出画成 R1 已实现的功能。

本章小结

读原理图要同时使用位号、引脚号和网络名。网络标签与电源符号压缩了图形，却不改变闭合回路；GND 是共同参考与返回网络。方向、有效电平、高阻、NC 与 DNP 都是设计语义，不能靠外观猜测。先按功能块追踪电源、信号和控制，再核对单个引脚，能显著减少误读。

本章事实依据：R1 最终项目归档中的网络与引脚、project-facts.json 的连接器和 MCU 映射，以及制造检查清单的 48/48 元件、172/172 引脚和 9 个有意未连接引脚记录。
