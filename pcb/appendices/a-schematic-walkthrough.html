    A.1 导读范围与证据边界

    本附录按“电源从哪里来、信号经过什么、控制由谁发出、故障从哪里返回”的顺序，完整导读 USB GPIO R1 原理图。最终设计包含 48 个 PCB 位号、172 个焊盘和 47 个命名网络；其中 J2、J3 保留在 PCB、CPL 和装配图中，但采购 BOM 明确不装配。

    事实状态：位号、标称值、封装、引脚网络和 DNP 属性来自最终 EasyEDA 项目归档；连通性、数量和制造几何来自发布清单与制造审核。当前没有装配板实测数据，因此电压、纹波、温升、USB 枚举、时钟起振和限流动作均不能写成“已测得”。

    阅读时应同时打开四种资料：原理图回答“逻辑上怎么连”，附录 B回答“每个实物是什么”，附录 C回答“连接器和 MCU 的针脚对应什么”，装配图与 PCB 回答“它实际放在哪里”。任何单一视图都不足以完成审核。

    A.2 一张原理图的六步阅读法

    1. 先找边界：J1 是主机侧入口，J2 是目标侧接口，J3 是调试接口。
    2. 再分电源域：VBUS_IN、3V3_SYS、5V_SW 和 3V3_SW 是四条不同电源网络，不能因电压相近而合并。
    3. 沿单向功能链追踪：例如 USB 数据依次经过 J1、U2、R3/R4，再到 U1；不要跨过中间保护器件只看两端。
    4. 为每个有源器件寻找供电、地、输入、输出和配置脚：缺少其中任意一类，都应视为待解释问题。
    5. 检查默认状态：R9、R10、R11、R12 与 C10 决定复位、启动和受控电源在固件接管前的倾向。
    6. 最后核对物理实现：原理图上相邻不代表 PCB 上相邻；去耦、ESD、晶振和 Buck 回路必须在布局中再次验证。

    快速自检：如果无法用一句话说明某条网络的源端、负载端和返回路径，就还没有真正读懂该网络。

    A.3 USB-C 入口、CC 检测与 VBUS

    A.3.1 J1：主机连接边界

    J1 是 USB 2.0-only Type-C 母座。重复的 A/B 侧 VBUS 焊盘汇入 VBUS_IN，重复的地焊盘和四个外壳/固定焊盘接入 GND；A6/B6 合并为 USB_DP_CONN，A7/B7 合并为 USB_DM_CONN。A8、B8 在本设计中刻意不连接。J1 左向伸出板边是最终机械意图，不是位置错误。

    A.3.2 R1、R2：两个独立的受电端声明

    R1 将 CC1 以 5.1 kΩ 下拉到地，R2 对 CC2 做同样处理。两只电阻不能合并成一只，也不能把 CC1 与 CC2 短接；可翻转插头只会使其中一条 CC 通道成为有效检测通道。开路或错值可能导致某一插入方向不能被主机识别。

    A.3.3 C1 与 VBUS 分配

    C1 是 VBUS_IN 的 10 µF 本地储能/旁路电容。VBUS 随后分成三条功能路径：给 U2 提供钳位参考、给 U3 产生始终开启的 3V3_SYS、给 U4 产生受控的 5V_SW。这些支路共享同一来源，但其局部电流回路和旁路电容不同。

    R1 不是 USB Power Delivery 设备，也没有隔离。5V_SW 是受控电源输出，不是 5 V 逻辑电平；可用电流还受主机、线缆、板上负载、开关压降和热条件限制。

    A.4 USB 数据保护链

    USB 数据路径应成对阅读，并保持 D+、D− 的拓扑对称：

    USB 数据从连接器到 MCU 的完整路径
    通道连接器侧ESD 器件串联电阻MCU 侧
    D−J1 → USB_DM_CONNU2 pin 1 → pin 6，网络变为 USB_DM_ESDR3，22 ΩUSB_DM_MCU → U1 pin 21
    D+J1 → USB_DP_CONNU2 pin 3 → pin 4，网络变为 USB_DP_ESDR4，22 ΩUSB_DP_MCU → U1 pin 22

    U2 的 pin 2 接地、pin 5 接 VBUS_IN，构成双线 ESD 导流拓扑。U2 必须靠近连接器，使“尚未受保护”的走线尽量短；R3、R4 则位于 ESD 级和 MCU 级之间，用于源端阻尼/调节。最终 CAD 几何显示三段 D+/D− 长度高度匹配，但这只证明几何长度，不证明生产叠层下的受控阻抗、眼图、EMI 或 ESD 合规。

    常见误读：看到 D+、D− 最终都到 U1，就跳过 U2 和 R3/R4。这样会漏掉保护方向、分段网络名以及器件错装后“仍有铜连接但 USB 不工作”的故障模式。

    A.5 5 V 到 3.3 V 的 Buck 电源

    U3（SY8089AAAC）、L1、C2、C3、R5 和 R6 构成始终开启的 3.3 V 电源：

    1. VBUS_IN 同时进入 U3 pin 1、pin 4；C2（10 µF）在输入侧向地旁路。
    2. U3 pin 3 输出高速切换的 BUCK_SW，经 L1（2.2 µH）传递能量。
    3. L1 后端形成 3V3_SYS，C3（22 µF）提供输出储能和纹波滤波。
    4. R5（453 kΩ）从 3V3_SYS 接到 BUCK_FB，R6（100 kΩ）再接地；按典型 0.6 V 反馈基准计算，标称输出约为 3.318 V。
    5. U3 pin 2 接地，输入开关环、开关节点、输出环和反馈返回都必须在 PCB 上保持紧凑且彼此合理隔离。

    BUCK_SW 不是“3.3 V 走线”，而是高 dv/dt 开关节点。其铜面积过大、靠近 USB/晶振/反馈网络，或输入输出回路过长，都可能造成纹波、EMI、温升或稳定性问题；这些问题不会因 ERC/DRC 为零而自动消失。完整公式见附录 D。

    A.7 MCU 供电、去耦与核心控制

    U1 为 CH32V203K8T6。pin 1、5、17 接 3V3_SYS，pin 16、32 接 GND。C6、C7、C8 各为 100 nF 高频去耦，C9 为 10 µF 区域储能。它们在原理图上都只是“电源到地”的并联电容，但在 PCB 上服务的瞬态路径不同；最终 CAD 最短路径记录为 C6/C7/C8 到 MCU 约 6.65/6.71/8.99 mm，C9 约 13.82 mm，这些是几何计算而非噪声实测。

    U1 同时承担 USB 设备、八路 GPIO、四路 TIM3 PWM、目标 UART、两路电源使能、状态 LED 和 SWD。因多个功能共享同一 MCU，固件必须在复位、启动和 USB 挂起期间建立安全状态：外部 GPIO 高阻、PWM 停止、UART TX 未配置前高阻、两路受控电源关闭。该行为目前属于预期要求，尚无板上测量证据。

    A.8 时钟、复位、启动与状态灯

    A.8.1 Y1、C11、C12

    Y1 的最终项目值是 12 MHz、标称负载 20 pF，覆盖了早期提案中的 8 MHz。Y1 pin 1/3 分别接 OSC_IN/OSC_OUT，pin 2/4 接地；C11、C12 各 12 pF，从两条振荡网络分别接地。只计算两只外接电容的串联等效得到 6 pF，仍须加上引脚、封装和走线杂散电容；因此负载匹配、频偏和起振裕量都标为待验证。

    A.8.2 R12、C10 与 R11

    R12（10 kΩ）把 NRST 上拉到 3V3_SYS，C10（100 nF）把 NRST 对地滤波，名义 RC 时间常数为 1 ms。实际复位释放时间还取决于 MCU 阈值和电源上升斜率。R11（100 kΩ）把 BOOT0 默认拉低，减少误入启动模式的风险。

    A.8.3 R13 与 D1

    U1 pin 28 的 STATUS_LED 经 R13（1 kΩ）到 LED_A，再经过黄绿色 LED D1 到地。由于最终源未给出板上 GPIO 高电平和 LED 正向压降的实测值，只能写成 ILED = (VOH − VF)/1 kΩ；令 VF=0 的保守数学上限为 3.3 mA，不是实测电流。

    A.9 GPIO、PWM、UART 与目标接口

    U1 的十条外部数字信号先经过 220 Ω 串联电阻，再到 J2。电阻把 MCU 侧网络与连接器侧网络有意分开：

    MCU 到目标接口的信号链

    TIM3_CH2 / GPIO1GPIO1_MCUR15GPIO1_PWM1 / pin 6
    TIM3_CH3 / GPIO2GPIO2_MCUR16GPIO2_PWM2 / pin 7
    TIM3_CH4 / GPIO3GPIO3_MCUR17GPIO3_PWM3 / pin 8

    USART2_RXUART_RX_MCUR23UART_RX / pin 14

    四路 PWM 共用 TIM3 频率，但可有独立占空比。UART 是 3.3 V CMOS/TTL，不是 RS-232。R14–R23 可限制部分故障电流并与线路电容形成阻尼，但不能把 GPIO 变成隔离、5 V 兼容或大电流输出；最终板也没有外部 GPIO ESD 阵列。

    A.10 J2、J3 与调试恢复路径

    J2 为 2×8、2.54 mm 目标排针，提供两地、两路受控电源、八路 GPIO/PWM、UART、NRST 和一个保留空脚。J3 为 1×5、1.27 mm SWD 接口，提供 3V3_SYS、GND、SWDIO、SWCLK、NRST。二者均为 DNP：裸板上有孔和铜，装配 BOM 中没有排针。

    调试器接线必须以接口针脚表为准，不能凭连接器外观猜方向。J3 的 3.3 V 脚用于电平参考/按既定调试方案连接，不应在未确认供电方向时让两个电源互相驱动。完整针脚表见附录 C。

    A.13 三条端到端复述练习

    1. USB 枚举链：J1 的 CC1/CC2 由 R1/R2 声明受电端；VBUS 形成 VBUS_IN；D+/D− 经 U2 和 R3/R4 到 U1；U1 还必须获得稳定的 3V3_SYS、时钟和复位释放。任何一环失败都可能表现为“不枚举”。
    2. 5 V 输出链：J1 VBUS → VBUS_IN → U4 → 5V_SW → J2 pin 3；U1 pin 26 通过 EN_5V 控制 U4，R9 提供默认关闭，R7 设置设计限流，C13/C4服务输入输出瞬态。
    3. PWM0 链：U1 pin 12 的 TIM3_CH1 → GPIO0_MCU → R14 → GPIO0_PWM0 → J2 pin 5；返回电流必须经目标与本板的共同 GND 回到驱动源。

    完成标准：能够不依赖原理图注释，按上述方式复述每个电源域、每条外部信号和每个控制反馈环，并能指出哪些结论仍须实物验证。
