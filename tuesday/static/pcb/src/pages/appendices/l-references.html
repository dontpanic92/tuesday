L.1 来源优先级与引用方法

本教程使用以下冲突解决顺序：最终 EasyEDA 项目归档 → 制造 manifest → fabrication checklist → 最终 BOM → 最终 CPL → 最终原理图/装配图/Gerber → 器件数据手册 → 早期提案 → 教程目录与实施要求。较早文档用于解释设计意图，但不能覆盖最终源中的位号、值、封装、网络或 DNP 状态。

引用标签：正文中的 final-project-archive、fabrication-manifest 等键对应下表。引用一个来源只表示该来源支持所述范围；例如 CAD 清单可支持几何和连通性，不能支持板上纹波或温度。

  • 设计文件和发布文件用于回答“最终 R1 实际采用什么”。
  • 数据手册用于回答“器件在什么条件下如何工作”。
  • 实验记录用于回答“这块装配板实际表现如何”；当前尚无此类结果。
  • 任何转载数据手册都应与制造商当前正式版本交叉核对。

L.6 延伸阅读的正确入口

  • USB：查 USB-IF 官方 USB 2.0、Type-C 与测试资料的当前版本；只使用与 USB 2.0 设备/受电端直接相关的章节，并记录版本。
  • PCB 制造：使用实际板厂的层叠、最小线宽/间距、孔、板边、阻焊、钢网和阻抗能力，而不是通用“经验值”。
  • 装配：查器件制造商的封装、回流焊、湿敏和返修指南，以及装配厂的 DFM 要求。
  • 测量：查仪器厂商关于短地弹簧、差分探测、带宽限制、电流测量和电子负载动态模式的应用资料。
  • EasyEDA：查与当前客户端版本匹配的官方项目、ERC/DRC、铺铜、Gerber 和制造输出文档；界面名称变化时以操作目的和输出证据为准。

延伸阅读只能补充方法，不能覆盖最终项目源。若新资料导致设计事实改变，应创建新板修订并重新发布，而不是直接改写 R1 的历史记录。
