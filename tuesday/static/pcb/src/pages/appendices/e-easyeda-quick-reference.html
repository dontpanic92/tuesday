E.1 适用范围与版本

本速查表面向 EasyEDA 中的项目组织、原理图、PCB、检查和发布流程。USB GPIO R1 最终项目归档记录的编辑器版本为 3.2.166；不同语言、版本和工作区的菜单名称可能变化，因此本页使用“操作目的 + 成功证据”，不编造固定菜单路径或快捷键。

核心原则：界面上“看起来正确”不是发布证据。每一步都要知道由哪一种检查证明：ERC 证明原理图规则，DRC 证明配置过的 PCB 规则，机械审核证明实体与方向，Gerber 查看器证明实际制造几何。

E.3 放置器件前的库核对

  1. 用完整可订购型号搜索器件，不只按通用名称选“长得像”的符号。
  2. 核对符号针号、针名和电气类型；对 MCU、USB-C、ESD、稳压器和负载开关逐针比对数据手册。
  3. 核对 footprint 的焊盘号、间距、孔径、外形、pin 1、极性、连接器插接方向和板边伸出量。
  4. 核对制造商料号、供应商代码、封装和 DNP 属性；把“加入 BOM”和“加入 PCB”视为两个不同决定。
  5. 对关键器件保存本地或项目级可追溯版本，避免公共库条目更新后静默改变设计。

不要这样做：仅凭 3D 模型或缩略图判断 footprint 正确。3D 外形、铜焊盘、孔和针号可以彼此不一致。

E.5 ERC 与原理图—PCB 同步

  1. 运行严格 ERC，按“错误、警告、已解释例外”分类；不要批量忽略。
  2. 人工复核 ERC 不理解的内容：USB-C 重复针、MCU 复用、默认状态、器件数值和符号—footprint 针号。
  3. 确认电源输出冲突、未驱动电源、悬空输入、意外 no-connect 和网络名拼写。
  4. 同步到 PCB 前保存；同步后查看新增、删除、网络变化和 footprint 变化清单。
  5. 核对组件、针脚、焊盘和网络数量。R1 发布基线为 48 个组件、172 个映射针脚/焊盘、47 个命名网络、原理图与 PCB 网络差异为 0。

R1 发布时严格 ERC 为 0 错误、2 个非阻塞提示；API 未暴露提示文本，因此教材只记录其存在，不能虚构内容。

E.7 布局速查

  1. 机械优先：先放 J1、J2、J3 和板边约束；验证插拔、排线、探针和外壳空间。
  2. 入口保护：U2 靠近 J1，缩短未保护的 USB 段；CC 电阻靠近连接器。
  3. 电源回路：U3、C2、L1、C3 紧凑；限制 BUCK_SW 面积，反馈远离噪声。
  4. 本地去耦：电容靠近所服务的电源脚，而不是只排得整齐。
  5. 时钟：Y1、C11、C12 靠近 U1，短、对称、无支路，避开开关节点。
  6. 重复通道：R14–R21 按 GPIO 顺序排列，保留可读性和一致间距。
  7. 机械审核：同时检查实体包络、courtyard/组件形状、板边、连接器方向和高度；电气 DRC 不能替代。

R1 的历史失误包括 J1 朝向、R1 与连接器实体关系、C8/C9 重叠；最终 1,128 对组件的实体和 courtyard 检查均为 0 重叠。

E.9 铺铜的强制四步规则

  1. 优先使用可重铺的原生铺铜区域。设置正确网络、层、间距、热焊盘、优先级和孤岛策略。
  2. 检查边界。铺铜边界及其 bounding box 必须位于闭合板框的预期范围内；边界不得越板或使用错误坐标系。
  3. 每次几何或规则变化后重铺。移动器件、改线、改板框、改间距或改层后都要 repour；不能相信旧填充。
  4. 在 Gerber 中确认实际铜。可见的边界对象不等于制造铜。导出后检查目标铜层中确实存在区域、间距、热连接和无孤岛。

R1 最终项目有 0 个原生 POUR、17 个连接到 GND 的静态 FILL（顶层 11、底层 6）。两个畸形越板原生铺铜边界已删除。静态填充不会自动随规则重算；今后任何 PCB 几何变化都必须重新生成并完整审核，未来版本应优先改用经验证的原生铺铜。

E.10 最终检查、状态刷新与发布

  1. 保存原理图和 PCB；关闭并重新打开 PCB，以刷新 EasyEDA 的连接图和活动文档状态。
  2. 运行最终 ERC、DRC，并分别记录 connection、clearance、differential-pair 和其他规则结果。
  3. 运行独立机械检查：实体、courtyard、板边、连接器方向、刻意伸出、探针和装配空间。
  4. 复核原理图—PCB：组件、针脚、焊盘、网络、DNP 和未连接项。
  5. 重铺/重建铜，检查所有电源、地和回流路径；再导出 Gerber。
  6. 导出并检查 BOM、CPL、装配图、原理图 PDF、Gerber/钻孔和项目归档。
  7. 用独立 Gerber 查看器检查铜、阻焊、锡膏、丝印、板框、PTH、NPTH 和过孔钻孔。
  8. 生成 manifest：版本、数量、尺寸、规则结果、文件列表、SHA-256 和未完成的实物验证。

变更失效规则：原理图、PCB、footprint、BOM、CPL 或制造输出有任何变化，旧签核全部失效；必须从受影响的最早阶段重新执行到最终 Gerber 与哈希。
