学习目标与安全边界

本章解释 8 路 GPIO、4 路共享 TIM3 的 PWM、一路 3.3 V TTL/CMOS UART，以及 R14–R23 与 J2 的接口保护作用。

关键边界：R1 的外部接口是 3.3 V 逻辑、与 USB 主机共地、无隔离；最终板没有外部 GPIO ESD 阵列。220 Ω 串联电阻可限制部分故障电流和边沿，但不是电平转换器、保险丝或完整 ESD 钳位。

事实状态：引脚映射和值为【设计】；公式算例为【计算】；逻辑阈值、边沿、最大波特率、负载能力和抗扰度为【待验证】。

13.1 数字输入阈值与数字输出电平

数字输入并不是“高于 0 V 就是 1”。器件数据手册会规定输入低电平上限和输入高电平下限；两者之间是不可依赖的过渡区。数字输出也不是理想电压源：负载电流增加时，高电平会下降、低电平会上升。

R1 将 GPIO、PWM 与 UART 定义为 3.3 V CMOS/TTL 逻辑【设计】。连接目标板前必须确认：

  • 目标输出高/低电平落入 U1 的推荐输入范围；
  • U1 输出电平满足目标输入范围；
  • 双方共地且没有把 5V_SW 当作 5 V 逻辑；
  • 目标在 R1 断电时不会经信号线反向供电。

事实库没有给出本板工作点的输入阈值或输出驱动曲线，因此不能在这里虚构具体阈值或允许电流。

13.2 推挽、开漏、上拉、下拉与高阻

推挽输出
主动拉高也主动拉低，边沿快，适合普通数字输出；两个推挽输出互接且电平相反会形成冲突电流。
开漏输出
只主动拉低，释放时依靠外部或内部上拉变高；可实现线与，但上拉阻值与电容决定上升时间。
输入上拉/下拉
为开关或开路信号提供默认状态；内部弱拉不能替代所有外部抗噪设计。
高阻态（Hi-Z）
引脚不主动驱动高或低。R1 要求复位、启动和未配置期间外部 GPIO 与 UART_TX 保持高阻【预期】。

改变模式时应先设置安全输出值，再切换方向或复用功能，避免瞬时毛刺。目标板断电时尤其应优先高阻。

因此 220 Ω 是故障限流、振铃抑制和速度之间的折中。它不能钳住 ESD 电压，也不能让 5 V 信号自动变成 3.3 V。

13.6 PWM 的频率、占空比、分辨率与平均效果

PWM 在每个周期内只输出高或低电平。频率 f 的单位为 Hz，周期 T = 1/f；占空比 D 是高电平时间占周期的比例，范围 0–1 或 0%–100%。

定时器时钟与周期计数共同决定频率和分辨率。频率越高，固定计数时钟下可用的占空比分级通常越少。实际频率、抖动和负载波形必须用示波器或逻辑分析仪验证。

13.7 UART 的方向、波特率、帧格式与共地

R1 的目标 UART 是 3.3 V TTL/CMOS 电平【设计】：

  • U1 PA2/USART2_TX → R22 → J2-13 UART_TX，方向为板到目标；
  • J2-14 UART_RX → R23 → U1 PA3/USART2_RX，方向为目标到板；
  • 双方必须共地，并配置相同波特率、数据位、校验和停止位。

UART 空闲电平、帧格式和实际最高可靠波特率必须由固件配置与实物测试确认。R22/R23、线缆电容和目标输入会共同改变边沿。

13.8 TTL UART 不是 RS-232

TTL/CMOS UART 以 0–3.3 V 附近的逻辑电平工作，并直接参考 GND。传统 RS-232 使用不同的电压极性与幅度，需要专用收发器。把 J2 的 UART_TX/RX 直接接到真正 RS-232 端口可能损坏 U1【预期】。

连接前不要只看“都是串口”：

  • 确认接口标为 3.3 V TTL/CMOS，而非 RS-232、RS-485 或 5 V TTL；
  • 交叉连接 TX→RX、RX←TX，并连接 GND；
  • 若目标电平不同，使用经过验证的电平转换器；
  • 长线、工业环境或不同接地系统应使用更合适的差分/隔离接口。

13.10 GPIO 不应直接驱动马达、继电器或大电流 LED

这些负载超出逻辑引脚的用途：

  • 马达和继电器需要远大于 GPIO 的电流，并产生感性反冲；
  • 高功率 LED 需要受控电流，不能只靠 GPIO 内阻；
  • 长线和外部接口会引入 ESD、浪涌和地电位差；
  • R14–R23 只能串联限流，最终 R1 没有外部 I/O ESD 阵列【设计事实】。

正确做法是使用逻辑级 MOSFET/驱动器、续流二极管、独立电源与必要的隔离/保护，并把地、电流回路和关断状态一起设计。5V_SW/3V3_SW 可给外部小负载供电，但也不能替代负载驱动级。

13.11 实验：LED、按键、PWM 与 UART 回环

准备与安全

  • J2 默认 DNP；使用方向已核对的排针或夹具，并在断电时接线。
  • 外接 LED 必须串联限流电阻；按键只接 3.3 V/GND 逻辑，不接 5 V。
  • UART 回环连接 J2-13 与 J2-14，并共用本板地；不要接 RS-232。
  • 不要接马达、继电器或未知目标板；出现过流、复位、异常发热立即断电。

A. 数字输出与 LED

  1. 断电连接一个带限流电阻的低电流 LED 到选定 GPIO。
  2. 上电后先保持高阻，再配置推挽输出并交替高/低。
  3. 记录 GPIO 电压、LED 状态和供电电流；不要预设亮度。

B. 数字输入与按键

  1. 用按键把选定输入接到 GND，并配置合适上拉。
  2. 记录按下/释放读数以及机械抖动；软件去抖时只改变一个参数。

C. PWM

  1. 在 GPIO0–GPIO3 中选择一路 TIM3 通道，从低频、低风险设置开始。
  2. 用示波器测频率、高低电平时间和占空比，再计算平均值。
  3. 同时启用第二通道，验证两路频率相同而占空比可不同。

D. UART 回环

  1. 断电短接 J2-13 UART_TX 与 J2-14 UART_RX。
  2. 配置一致的 UART 帧格式，发送已知字节序列并比较接收结果。
  3. 逐步改变波特率或线缆条件，记录首次错误条件；不要直接跳到未经验证的最高速度。

本章小结

  • R14–R23 全部为 220 Ω，逐一隔开 U1 侧与 J2 侧的 8 GPIO/PWM 和 UART TX/RX。
  • 前四路共享 TIM3 频率而可独立设置占空比；PWM 平均效果取决于负载。
  • UART 是 3.3 V TTL/CMOS，不是 RS-232；双方必须共地并交叉 TX/RX。
  • J2-15 是 NRST，J2-16 保留；最终板没有外部 GPIO ESD 阵列或隔离。
