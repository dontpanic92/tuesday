学习目标与位号校正

本章解释两只 SY6280 负载开关如何让 U1 用低电流逻辑控制 5V_SW 与 3V3_SW，并分析默认关闭、可调限流、输入/输出电容、反向供电和共享 USB 预算。

位号校正：toc.md 的 10.4 写成“R6 和 R9”。最终 R1 的两只使能下拉是 R9/R10；R6 是 U3 反馈下臂。限流设定为 R7/R8。以下内容按最终工程源。

事实状态：网络、标称值和目标电流为【设计】；0.40237 A 与 0.25185 A 为【计算】；限流容差、压降、热关断、恢复和真实 USB 预算为【待验证】。

10.1 为什么不能让 MCU 引脚直接开关负载电流

MCU GPIO 的任务是传送逻辑状态，不是承载数百毫安电源电流。若把目标负载直接接到 U1 引脚，输出晶体管、封装焊线和 3V3_SYS 都可能过流，感性负载还会产生反向电压。R1 让 U1 只驱动 EN_5V 与 EN_3V3，大电流则通过 U4/U5 的高侧功率通道。

这种分工带来四个好处：逻辑与功率路径分离、上电默认可控、可设置电流限制、过载时由专用器件承担热应力。但它不等于完全防误接；目标端仍与主机共地，且最终设计没有隔离、外部 GPIO ESD 阵列或负载开关故障输出。

10.3 5V_SW 与 3V3_SW 不是同一电源

两路受控电源的来源与风险
电源上游控制器主要约束
5V_SWJ1 的 VBUS_INU4共享 USB 主机、线缆、连接器和全板输入电流预算。
3V3_SWU3 的 3V3_SYSU5同时受 U3 转换能力、3V3_SYS 压降、U5 热和全板 3.3 V 负载约束。

5V_SW 是电源输出，不是 5 V GPIO 逻辑电平。R1 的 GPIO、PWM 与 UART 均为 3.3 V 逻辑【设计】。把 5V_SW 接到任一 MCU I/O 可能损坏 U1。

10.7 浪涌、短路、限流与热关断

当负载电流试图超过设定值时，U4/U5 预计通过调节内部开关限制电流；持续过载会产生功耗 P ≈ VDROP × I，温度上升后可能进入热保护【预期】。恢复方式、阈值分散和自动重试行为必须按实际器件资料及板上试验确认。

限流并不消除能量：

  • 5V_SW 短路时，U4、VBUS 走线、J1 和主机端口承受压力；
  • 3V3_SW 短路时，U5 之外还会加载 U3，可能拖低 U1 的 3V3_SYS；
  • 热关断只保护器件达到其机制边界，不保证目标板、线缆或接头不会过热。
不要用金属线直接短路做“测试”。使用电子负载或额定功率电阻，在限流电源下逐步逼近，并由教师或具备资质的工程人员监督故障测试。

10.8 反向供电与未上电目标板

反向供电（back-powering）是指电流从信号或电源输出进入本应断电的目标板，再经其保护二极管、稳压器或内部电源网反向流动。常见危险场景包括：

  • 目标板自己有电，而 R1 的 5V_SW/3V3_SW 关闭，两端电源被直接并联；
  • 受控电源关闭，但某个 GPIO/UART 仍输出高电平，经目标输入保护结构供电；
  • 目标板把电压送回 J2 的电源脚，内部开关是否允许反向电流未被验证。

因此上电顺序必须明确：先共地、核对电压域，再决定由哪一侧供电；在关闭目标电源前，GPIO/PWM/UART 应先回到高阻或安全状态【预期】。R1 没有隔离，不能靠软件完全消除错误接线风险。

10.9 USB 总电流、压降与热预算

若只为说明数量级，采用事实库记录的“80 mΩ 级”开关电阻并忽略其他路径，U4 在 0.4 A 时开关本体压降约为 0.4 × 0.08 = 0.032 V，U5 在 0.25 A 时约为 0.020 V【理想化计算】。这不是保证值：温度、器件分散、PCB、J1、J2 和线缆都会增加压降。

USB 输入还要同时供应 U1、U3 损耗、LED、两路开关静态消耗和外部负载。实际主机/线缆能力、总输入电流和热平衡为【待验证】。

10.10 实验：安全测试使能、过载、压降与恢复

前提与器材

仅在 U3、U1、固件安全状态和 J2 接线已先验证后进行。器材：限流 5 V 电源、USB-C 供电夹具、电子负载、万用表、示波器、温度测量工具及已核对方向的 J2 夹具。

安全与停止条件

  • 首次测试不使用电脑 USB 口作为故障能量源。
  • 先把电子负载设为 0 A；确认共地、极性和两路输出默认关闭。
  • 出现 3V3_SYS 下跌、U1 复位、器件快速升温、异味、波形持续振荡或输入限流时立即断电。
  • 不允许用导线硬短路；不要同时改变负载、电源限制和固件状态。

步骤

  1. 复位或保持 U1 高阻，确认 5V_SW、3V3_SW 为关闭状态；记录而非假定。
  2. 分别使能 U4、U5，在轻载下测输入与输出电压，计算路径压降。
  3. 逐步增加负载，每一步等待稳定并记录电流、电压、温度和是否复位。
  4. 在经批准范围内接近预期限流，观察输出电压和电流；到达停止条件立即退回。
  5. 撤去过载，记录能否恢复、恢复时间和是否需要重新使能。
  6. 关闭输出后记录电压衰减；不要把衰减误称为“集成放电功能”，除非有相应证据。

本章小结

  • U4 从 VBUS_IN 产生 5V_SW，U5 从 3V3_SYS 产生 3V3_SW；U1 只控制 EN。
  • R9/R10 使两路默认关闭，R7/R8 分别计算得到约 0.402 A 与 0.252 A 的典型限流。
  • C13/C14 服务输入高频回路，C4/C5 服务输出浪涌和负载阶跃。
  • 反向供电、共享 USB 预算、压降、限流和热恢复均需受控实验验证。
