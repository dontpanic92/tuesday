本章读法与完整性声明

本章把 R1 的 48 个 PCB 物理位号逐一串起来：R1–R23、C1–C14、U1–U5、J1–J3、L1、D1、Y1。每项均给出最终值或型号、网络、功能、选值理由、布局理由、失效影响和证据状态。

  • 【设计】最终 EasyEDA 工程明确采用的值、网络、型号或 DNP 状态。
  • 【计算】由公式或 CAD 数据得到，不是仪器测量。
  • 【预期】依据机理推断的失效现象。
  • 【待验证】没有装配板证据；当前事实库的实测条目为 0。

表格中的“开路/短路”是诊断假设，不是建议人为制造的故障。涉及电源、USB、晶振或过流的验证必须使用限流、合适探具和明确停止条件。

14.1 如何联合使用 BOM、原理图、PCB 与装配图

  1. 从位号开始：例如 R7 是一个实例，不应只搜索“16.9 kΩ”。
  2. 在原理图确认网络：看它的每个引脚连接到哪里，以及信号/电源流向。
  3. 在最终工程确认值和型号：R1 的 BOM CSV 的 Value 列为空，不能把空白误读为“未知值”；最终值来自项目归档的 DEVICE/实例元数据。
  4. 在 PCB/CPL 确认位置、方向和装配面：全部 48 个物理位号在顶层；CPL 含 48 行。
  5. 在 BOM 确认是否装配：46 个位号拟装，J2/J3 为【设计-DNP】，因此不在采购 BOM，但仍在 PCB、CPL 和装配图中。
  6. 用装配图定位：板上丝印为减少拥挤而隐藏密集位号，装配 PDF 和 CPL 才是定位依据。
  7. 最后核对状态：CAD 连通、值和位置正确不等于电压、温升、USB 或时钟已经实测。

14.9 L1：Buck 能量存储元件

本章小结

R1 的 48 个物理位号形成一条完整因果链：J1/R1/R2 建立 Type-C 受电连接；U2/R3/R4 处理 USB 数据入口；U3/L1/C2/C3/R5/R6 生成 3V3_SYS；U4/U5 及其阻容控制两路电源；U1/C6–C12/R11/R12/J3 构成控制核心；R13/D1 提供状态；R14–R23/J2 把 GPIO、PWM、UART 和电源带到目标端。任何“省略一件仍然一样”的判断，都必须经过网络、机理、故障和实物证据四重检查。

事实来源

本章以最终 EasyEDA 项目归档为位号、值、器件、网络和 DNP 的最高权威，并与最终 BOM、CPL、原理图 PDF、装配 PDF、fabrication manifest/checklist 及 component-atlas.json 交叉核对。BOM 空 Value 列未被解释为未知值；J2/J3 的 DNP 状态未被误写成缺件；所有硬件性能仍按【待验证】处理。
