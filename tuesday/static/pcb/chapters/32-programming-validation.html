    学习目标与验证边界

    本章把“电源能够稳定建立”推进到“MCU 可识别、可编程、能以安全状态启动，并能按固件定义枚举为 USB CDC 设备”。编程成功只证明调试链路和部分 MCU 功能，不证明 USB 信号完整性、GPIO 负载能力、Buck 纹波或长期可靠性。

    R1 尚无实物编程或 USB 枚举记录。下列现象均是设计要求或预期结果。使用已经评审、可追溯的实验固件二进制及其哈希；本任务不修改板、固件或制造文件，也不把任何未执行步骤写成已通过。

    建议按以下质量门顺序操作：限流供电稳定 → SWD 识别 → 备份与擦写 → 下载后校验 → 安全状态 → LED → USB CDC → 复位/恢复 → 版本报告。

    32.1 安全连接 SWD

    J3 是 1×5、1.27 mm 的调试接口，PCB 上存在但默认 DNP。使用焊接排针或弹簧针治具时，先用装配图和万用表确认 pin 1；不要仅凭线色或连接器缺口猜方向。

    表 32-1　J3 调试接口
    J3 引脚网络用途注意
    13V3_SYS目标电压参考/供电连接，取决于调试器模式默认由板端限流供电；禁止与调试器电源输出争用
    2GND共同参考最先连接、最后断开
    3SWDIO双向调试数据3.3 V 逻辑
    4SWCLK调试时钟线短、与地回路紧邻
    5NRST硬件复位/连接恢复只能用开漏或兼容驱动拉低

    推荐接线顺序

    1. 关闭板端限流电源与调试器输出，确认 3V3_SYS 已降到 0.2 V 以下。
    2. 把调试器设为“不向目标供电”或“仅检测目标电压”；若工具没有该模式，查阅工具手册后选择唯一供电源。
    3. 依次连接 GND、目标电压参考、SWDIO、SWCLK、NRST；用通断档复核治具到 J3 的映射。
    4. 从板端限流电源上电，确认 3V3_SYS 与电流仍处于第 31 章基线范围，再启动调试软件。
    5. 从低 SWD 时钟开始；通信稳定后才提高速度。长飞线、无地回流和过高时钟都会造成偶发错误。

    若接上调试器后板端电流明显改变、3V3_SYS 被抬高、调试器发热或电源互相反灌，立即断电。不要通过反复插拔带电排针“碰运气”。

    32.2 读取器件身份并写入 Flash

    U1 的最终设计型号为 CH32V203K8T6。丝印、采购记录和调试器读取值应共同支持同一器件身份；本事实库没有给出可安全抄写的数值器件 ID，因此必须由实际工具读取并与当前数据手册/编程工具数据库核对。

    1. 记录板号、调试器型号/固件、编程软件版本、SWD 时钟和目标供电电压。
    2. 执行“连接并停机”，读取芯片族、器件 ID、Flash 容量、保护/选项字节；先保存截图或文本日志。
    3. 若允许且流程需要，读取出厂内容或至少保存选项字节；不要未经记录就执行全片擦除。
    4. 选择经批准的二进制，核对文件名、版本、构建时间、目标板版本和 SHA-256 哈希。
    5. 擦除、编程并执行逐字节或工具提供的 verify。写入成功但 verify 失败仍判为失败。
    6. 断开调试会话并冷启动一次，确认程序不是只在调试器保持复位/供电时运行。

    常见故障：目标未供电、J3 反接、SWD 时钟过高、NRST 被 C10/外部治具持续拉低、BOOT0 状态异常、芯片保护打开或焊接不良。一次只改变一个条件并保留日志。

    32.3 GPIO 安全初始化

    安全启动要求是：外部 GPIO 为高阻输入、PWM 停止、UART_TX 在配置前为高阻、5V_SW/3V3_SW 在复位、启动和 USB 挂起期间关闭。该要求来自设计意图，尚未在硬件上验证。

    固件应采用的先后顺序

    1. 最先把 PB3/EN_5V 与 PB4/EN_3V3 明确配置为低；R9/R10 的 100 kΩ 下拉只提供硬件默认，不替代固件。
    2. 把 PA6、PA7、PB0、PB1、PA0、PA1、PA4、PA5 对应的 8 路 GPIO 保持为输入或模拟高阻，禁止启动时产生脉冲。
    3. 保持 TIM3 CH1～CH4 停止并清除待输出比较状态，再按命令启用 PWM。
    4. 在设置 USART2 之前保持 PA2/UART_TX 高阻；配置完波特率和空闲电平后再切换复用功能。
    5. 最后初始化状态 LED 和 USB；错误路径、看门狗复位与挂起回调也必须回到同一安全状态。

    不依赖“看起来没事”的验证

    在 J2 上为待测引脚外接 100 kΩ 到 GND 或 3.3 V 的弱偏置，观察复位期间节点是否跟随外部偏置；能跟随说明更接近高阻，固定强驱动则提示固件或焊接问题。测试时每次只偏置一个引脚，外部电压不得超过 3.3 V，并保留 R14～R23 的 220 Ω 串联电阻但不要把它们当作过压保护。

    通过条件：上电、复位、下载和 USB 未配置阶段都没有可观察的受控电源误开启或 I/O 强驱动；实际电平、时间和测试阻值记录完整。

    32.4 USB CDC 枚举

    项目目标是 USB 2.0 Full-Speed 设备，并以 USB CDC ACM 提供虚拟串口。CAD 中 D+/D− 三段长度和偏斜已计算，但没有受控阻抗证明、眼图或实物枚举结果；几何接近不等于 USB 已验证。

    1. 先用限流电源确认板稳定，再断电并移除板端 VBUS 供电夹具；防止与电脑 VBUS 并联。
    2. 选用已知可传数据的短 USB-C 线，记录线缆编号、主机端口、操作系统和是否通过集线器。
    3. 连接主机，记录从插入到系统出现设备的时间、VID/PID、产品字符串、序列号、配置和 CDC 端口名；具体值应与固件发布说明一致，不在本章猜测。
    4. 打开串口但先不发送命令，确认不会因 DTR/RTS 或端口打开而意外使能输出。
    5. 发送固件定义的只读“身份/状态”命令，保存原始响应；不要先执行 GPIO 或电源控制命令。
    6. 安全弹出或关闭端口，重复冷插拔三次，并记录是否每次稳定重现。

    32.5 状态 LED 测试

    U1 的 PB5/STATUS_LED 经 R13 = 1 kΩ 接 D1 阳极，D1 阴极接 GND，因此设计上为高电平点亮。按 ILED = (VOH − VF)/1 kΩ；若把 LED 正向压降极端地取 0，3.3 V 下电流上界为 3.3 mA。真实 VOH、VF 和亮度尚未测量。

    1. 复位后先观察 LED 默认状态，确认固件不会把随机闪烁误当“成功”。
    2. 通过已定义的只读/测试命令依次执行灭、亮、1 Hz 闪烁；每种状态保持足够时间以便观察。
    3. 用 DMM 测 PB5 侧、LED_A 与 GND 的电压；必要时用示波器核对闪烁周期。
    4. 复位板，确认 LED 回到固件规定的安全默认状态。

    判定：命令、逻辑电平和可见状态一致为通过。LED 不亮但 USB/SWD 正常时，优先查 D1 方向、R13 开路、焊点和引脚配置，不把它直接判为 MCU 死亡。

    32.6 复位与启动恢复测试

    R12/C10 形成名义 1 ms 时间常数，R11 = 100 kΩ 把 BOOT0 默认拉低。真实复位释放时间和启动模式仍需实测；恢复步骤必须遵循 CH32V203 与所用编程器的当前文档。

    1. 热复位：保持 USB 连接，通过 J3 pin 5 将 NRST 短暂拉低；记录 CDC 是否断开、释放后是否重枚举，以及输出是否先回安全态。
    2. 冷启动：断开所有电源至轨电压低于 0.2 V，再重新连接；比较版本报告与热复位结果。
    3. 损坏应用恢复：使用已批准的“连接并保持复位”流程重新建立 SWD；不要随意改变选项字节。
    4. 启动模式：确认 R11 使 BOOT0 默认低。若必须使用 ROM 启动恢复，按器件文档施加临时、可撤销的 BOOT0 状态，并在完成后验证恢复默认低。

    32.7 固件版本与板版本报告

    “端口出现了”不足以证明刷入了正确程序。设备的只读身份响应应把实物、固件和测试记录绑定起来，且不能仅依赖操作者手写文件名。

    表 32-5　建议的机器可读身份字段
    字段示例格式用途
    协议版本protocol=major.minor主机工具兼容性
    固件版本/构建fw=…; build=…定位二进制与源码提交
    目标板版本board=USB_GPIO_R1防止 R1/R2 固件混用
    功能能力gpio=8; pwm=4; uart=1主机不靠猜测启用功能
    安全状态rails=off; pwm=stopped上位机连接前确认
    唯一标识（若政策允许）芯片 UID 的受控表示测试追溯；注意隐私和克隆策略

    若 R1 没有硬件版本电阻或 EEPROM，固件报告的 R1 只是“构建目标”，不能自动证明所插实物就是 R1；仍需与板丝印、批次号和测试治具扫码交叉核对。

    放行到接口实验：SWD 识别、Flash 校验、冷启动、安全初始化、LED、USB CDC、复位恢复和版本报告均有原始记录。任一项失败或固件哈希不明，板应保持隔离，不进入外部负载和过流实验。

    本章小结

    可编程只是起点。可靠的基础验证要求唯一供电源、正确 SWD 引脚、可追溯二进制、写后校验、安全 I/O 顺序、稳定 CDC 枚举以及可恢复的复位路径。所有 R1 实测栏仍待实验填写；下一章才对外部接口逐项施加受控激励和负载。
