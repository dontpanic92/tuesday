<h2><span className="num">A</span>数学速查：点积、矩阵乘、softmax 稳定性</h2>

<h3>点积</h3>

$$
a\cdot b = \sum_{i=0}^{d-1} a_i b_i
$$

点积给出两个向量逐元素乘积的总和。Attention score 通常由 query 与 key 的点积再按维度缩放得到。

<h3>矩阵乘 shape</h3>

$$
A\in\mathbb{R}^{M\times K},\quad
B\in\mathbb{R}^{K\times N}
\quad\Rightarrow\quad
C=AB\in\mathbb{R}^{M\times N}
$$

$$
C_{i,j}=\sum_{p=0}^{K-1}A_{i,p}B_{p,j}
$$

<h3>HuggingFace transposed-weight linear</h3>

HuggingFace checkpoint 中的线性层权重通常按`[out_features, in_features]`存储：

$$
Y = XW^\top,\qquad
X\in\mathbb{R}^{S\times K},\quad
W\in\mathbb{R}^{N\times K},\quad
Y\in\mathbb{R}^{S\times N}
$$

<h3>稳定 softmax</h3>

$$
p_i = \frac{\exp(z_i-m)}{\sum_j\exp(z_j-m)},\qquad
m=\max_j z_j
$$

对所有 logits 同时减去最大值不会改变 softmax 结果，可降低指数运算溢出的风险。

<h2><span className="num">B</span>Rust 系统编程与 FFI 速查</h2>

<h3>Ownership 与 GPU 资源</h3>

Rust ownership 可用于表达 GPU 资源的生命周期：一个`DmlBuffer`拥有对应的 COM resource handle；对象 drop 时释放引用计数。由于 GPU 命令异步执行，资源释放或复用必须晚于 GPU 对该资源的最后一次访问。

<h3><code>unsafe</code></h3>

调用 D3D12、DirectML 或 FFI 时需要使用`unsafe`。本教程采用的边界是：将`unsafe`限制在小范围内，对外提供安全接口，并记录调用者必须满足的前置条件。

<h3>C ABI</h3>

```rust
#[no_mangle]
pub extern "C" fn xinfer_version() -> *const c_char {
    ...
}
```

C ABI 设计遵循以下规则：

- 不要让 panic 跨过 FFI 边界；
- 用 opaque handle 隐藏 Rust 内部结构；
- 用错误码返回状态，用`last_error`提供消息；
- 谁分配，谁释放。

<h3><code>bytemuck</code> 与 <code>half</code></h3>

`bytemuck::cast_slice`用于在满足布局条件时把数值切片视为字节切片，便于上传到 GPU。`half::f16`与`half::bf16`用于加载或转换半精度权重。

浮点加法不满足严格结合律：

$$
(a+b)+c \ne a+(b+c)\quad\text{in floating point}
$$

因此，CPU 与 GPU 的归约顺序不同会产生微小数值差异。测试通常采用容差判断：

$$
\max_i |y_i^{gpu}-y_i^{cpu}| < \varepsilon
$$

<h2><span className="num">D</span>GPU 调试：PIX、D3D12 debug layer、device removed</h2>

<h3>D3D12 debug layer</h3>

D3D12 debug layer 可报告资源状态错误、非法绑定和命令列表使用错误。开发 GPU backend 时应尽早启用。

<h3>Device removed</h3>

当 GPU device 被移除或重置时，D3D12 调用可能返回类似`0x887A0005`的 HRESULT。调试步骤：

1. 打开 debug layer；
2. 调用`GetDeviceRemovedReason`；
3. 隔离到最小 shape / 最小 kernel；
4. 检查 resource states 与 bindings；
5. 必要时绕开有问题的库 operator，改写 custom kernel。

<h3>PIX</h3>

PIX 是 Windows 与 Xbox 平台的 GPU 捕获和分析工具，可检查 command list、resource state、shader 与 timing。正文不依赖 PIX；在复杂 GPU 问题和性能分析中应优先考虑使用。
