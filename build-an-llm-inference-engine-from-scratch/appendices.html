A数学速查：点积、矩阵乘、softmax 稳定性

点积

\[ a\cdot b = \sum_{i=0}^{d-1} a_i b_i \]

点积衡量两个向量方向的相似度。Attention 的 score 就是 query 与 key 的缩放点积。

矩阵乘 shape

\[ A\in\mathbb{R}^{M\times K},\quad B\in\mathbb{R}^{K\times N} \quad\Rightarrow\quad C=AB\in\mathbb{R}^{M\times N} \] \[ C_{i,j}=\sum_{p=0}^{K-1}A_{i,p}B_{p,j} \]

HuggingFace transposed-weight linear

权重常存为 [out_features, in_features]：

\[ Y = XW^\top,\qquad X\in\mathbb{R}^{S\times K},\quad W\in\mathbb{R}^{N\times K},\quad Y\in\mathbb{R}^{S\times N} \]

稳定 softmax

\[ p_i = \frac{\exp(z_i-m)}{\sum_j\exp(z_j-m)},\qquad m=\max_j z_j \]

减去最大值不改变结果，但能避免指数溢出。

BRust 系统编程与 FFI 速查

Ownership 与 GPU 资源

Rust ownership 很适合包装 GPU 资源：一个 DmlBuffer 可以拥有一个 COM resource handle； 当对象 drop 时，引用计数释放。需要注意的是 GPU 命令是异步的，不能在 GPU 尚未使用完资源时提前释放或重用。

unsafe

调用 D3D12 / DirectML / FFI 都会遇到 unsafe。本教程的原则是：把 unsafe 封装在小范围内， 对外提供安全函数，并在注释中说明前置条件。

C ABI

#[no_mangle]
pub extern "C" fn xinfer_version() -> *const c_char {
    ...
}

C ABI 规则：

  • 不要让 panic 跨过 FFI 边界；
  • 用 opaque handle 隐藏 Rust 内部结构；
  • 用错误码返回状态，用 last_error 提供消息；
  • 谁分配，谁释放。

bytemuck 与 half

bytemuck::cast_slice 用于把 &[f32] 视为 &[u8] 上传到 GPU。 half::f16 / half::bf16 用于加载或转换半精度权重。

C浮点：f32 / f16 / bf16、舍入与容差

格式字节特点本项目用途
f324精度较高，带宽占用大CPU reference、activation
f162尾数较少，显存减半GPU weights
bf162指数范围接近 f32，尾数更少checkpoint dtype，加载时转换

浮点加法不满足严格结合律：

\[ (a+b)+c \ne a+(b+c)\quad\text{in floating point} \]

因此 CPU 与 GPU 归约顺序不同会导致微小差异。测试通常使用：

\[ \max_i |y_i^{gpu}-y_i^{cpu}| < \varepsilon \]

DGPU 调试：PIX、D3D12 debug layer、device removed

D3D12 debug layer

Debug layer 可以发现资源状态错误、非法绑定、命令列表使用错误。开发 GPU backend 时应尽早打开。

Device removed

如果 GPU device 被移除或重置，D3D12 调用可能返回类似 0x887A0005 的 HRESULT。 调试步骤：

  1. 打开 debug layer；
  2. 调用 GetDeviceRemovedReason；
  3. 隔离到最小 shape / 最小 kernel；
  4. 检查 resource states 与 bindings；
  5. 必要时绕开有问题的库 operator，改写 custom kernel。

PIX

PIX 是 Windows / Xbox 上的 GPU 捕获与分析工具。它可以查看 command list、resource state、shader、timing。 本教程正文不依赖 PIX，但生产级调试非常推荐使用。
